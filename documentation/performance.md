# Performance

Initially, all items will be marked with a ❌. The main developer is responsible for completing these items. When an item is finished, change the mark to ✅. If you are unsure about something and need assistance from a more experienced person, use 🔔.

| Performance Item | Status | Team Responsable | Comments |
| ------ | ------ | ------ | ------ |
| **HTML Validator:** Check for HTML errors. When possible, ensure heading tags follow a proper hierarchy (H1, H2, H3…), and make sure every page includes exactly one H1. [link](https://validator.w3.org/) | ❌ | Frontend |  |
| **Keyboard accessibility & focus behavior:** All interactive elements must be reachable and usable using only the keyboard, in a logical and predictable order. Focus styles must be visible. | ❌ | Frontend |  |
| **Accessibility errors (PageSpeed):** There should be no accessibility-related errors flagged in PageSpeed | ❌ | Frontend |  |
